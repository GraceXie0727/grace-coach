# Mode 3：行业/概念学习

帮助用户系统搞懂一个行业怎么运作，或深入理解一个新概念。与 Mode 1/2 的关键区别：每章先联网搜最新信息。入库走 SKILL.md Phase 6 的统一规则（读书笔记 → `知识卡片/概念/`，个人判断 → `知识卡片/我的判断/`，复习笔记和核心概念清单同步更新）。

---

## Phase 1：确认课题

如果用户没给明确主题，先问：

> 你想了解哪个行业或概念？比如「AI 行业」「Web3」「供应链」。

新课题同样要问学习目的和背景授权（规则见 SKILL.md Mode 1 Phase 1），和确认主题合并成一次问。

确认后，创建以下文件（全部同时创建）：

```
课题/_行业概念/{主题}/00-学习计划.md
课题/_行业概念/{主题}/01.md
复习笔记/{主题}.md      ← 用 知识卡片/_模板/复习笔记.md
```

`00-学习计划.md` 包含：学习目标、「学习目的」「学习者画像」、章节安排（按下方通用框架微调）、当前进度、来源说明，以及一张空的「核心概念清单」表。

同步更新 `INDEX.md` 的「行业/概念学习」板块。

---

## Phase 2：通用章节框架

按主题微调，但通常包含以下章节：

| 章节 | 内容 |
|------|------|
| 01 | **边界**：这个行业/概念到底在解决什么问题，不解决什么 |
| 02 | **运作**：产业链、关键玩家、价值怎么流动 |
| 03 | **商业模式**：钱从哪来、谁赚谁的钱、利润在哪个环节 |
| 04 | **变量与趋势**：什么在变、关键不确定性、最新动态 |
| 05 | **机会点**：这个行业的变化对用户意味着什么（`00-学习计划.md` 的「学习目的」「学习者画像」有值就围绕它写；没有就写普通从业者和旁观者的常见切入点） |
| 06 | **整合**：用户的判断——回顾各章，写出自己的综合理解 |

如果是一个概念（而非行业），可以压缩为 3-4 章：概念边界 → 不同视角/争议 → 适用场景与反例 → 用户的判断。

---

## Phase 3：每章生成流程

### Step 1：联网搜索

**每章开头必须先用联网搜索** 搜该章主题的最新信息（2-3 个搜索查询）。

搜索策略：
- 查询 1：`{主题} {本章关键词} {今年} {去年}`（最新动态）
- 查询 2：`{主题} {本章关键词} 分析/报告`（行业分析）
- 查询 3（可选）：`{主题} {本章关键词} 争议/问题`（批判视角）

### Step 2：写章节

结合联网搜索结果和训练知识写章节。文章结构和末尾的「学习反馈」区与 SKILL.md Phase 4 完全一致，唯一区别是正文开头加一行来源标注：

> 本章信息综合自公开报道和行业分析（{搜索日期}联网获取）及 AI 训练知识。

### Step 3：更新计划和索引

与 SKILL.md Phase 5 相同：更新 `00-学习计划.md` 和 `INDEX.md`。

### Step 4：学习反馈入库 + 认知点评

与 SKILL.md Phase 6 + Phase 7 完全相同。下一章开头先自动入库，再输出认知点评。

Mode 3 唯一差异：卡片「来源」栏写 `行业/概念学习「{主题}」第 {序号} 篇（{日期}联网获取）`。
